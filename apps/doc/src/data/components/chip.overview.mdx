## Chip — aperçu et usage

Les Chips sont de petits composants interactifs qui représentent une action, un choix ou une entrée compacte. Ils suivent les principes de Material 3 et se déclinent en quatre grands types d’usage :

- Assist (déclenche une action rapide, proche d’un bouton)
- Filter (sélection on/off ou multi‑sélection)
- Input (représente une valeur saisie par l’utilisateur, ex. un tag)
- Suggestion (propose des options contextuelles)

Cette page explique comment utiliser `Chip` dans Udixio, avec des exemples pratiques. Le code du composant pourra évoluer, mais les concepts et bonnes pratiques restent valables.

## Installation et import

```bash
import { Chip } from "@udixio/ui-react"
```

## Exemple minimal

```tsx
<Chip label="Option" />
```

Vous pouvez aussi fournir le texte via les enfants :

```tsx
<Chip>Option</Chip>
```

## Variants visuels

Selon le contexte, vous pouvez ajuster le style visuel. Les variantes les plus courantes sont : `filled`, `elevated`, `tonal`, `outlined`.

```jsx
<div className="flex flex-wrap gap-3">
  <Chip variant="filled">Filled</Chip>
  <Chip variant="elevated">Elevated</Chip>
  <Chip variant="tonal">Tonal</Chip>
  <Chip variant="outlined">Outlined</Chip>
</div>
```

Astuce : utilisez `filled/tonal` pour un accent plus marqué, `outlined/elevated` pour une présence visuelle plus légère.

## Types d’usage

### Assist chip (action rapide)

Un assist chip déclenche une action immédiate : ouvrir une vue, copier un texte, appliquer un filtre temporaire, etc.

```jsx
<Chip label="Nouveau" icon={faPlus} onClick={() => console.log('nouveau')} />
```

### Filter chip (sélection)

Rendez un chip sélectionnable avec `onToggle`. Contrôlez l’état via `activated` si nécessaire.

```jsx
function Filters() {
  const [selected, setSelected] = React.useState({ jazz: false, rock: true });
  return (
    <div className="flex flex-wrap gap-3">
      <Chip
        variant="outlined"
        activated={selected.jazz}
        onToggle={(v) => setSelected((s) => ({ ...s, jazz: v }))}
      >Jazz</Chip>
      <Chip
        variant="outlined"
        activated={selected.rock}
        onToggle={(v) => setSelected((s) => ({ ...s, rock: v }))}
      >Rock</Chip>
    </div>
  );
}
```

### Suggestion chip (proposition)

Présentez des suggestions contextuelles (réponses rapides, filtres de recherche). Gardez des libellés courts (idéalement ≤ 20 caractères).

```jsx
<div className="flex flex-wrap gap-2">
  <Chip variant="elevated">Près de moi</Chip>
  <Chip variant="elevated">Ouvert maintenant</Chip>
  <Chip variant="elevated">4★ et +</Chip>
</div>
```

### Input chip (valeur saisie)

Représente des entrées d’utilisateur (ex. tags). L’ajout/retrait est géré par votre logique applicative.

```jsx
function Tags({ tags, onRemove }) {
  return (
    <div className="flex flex-wrap gap-2">
      {tags.map((t) => (
        <Chip key={t} variant="tonal" label={t} onClick={() => onRemove(t)} />
      ))}
    </div>
  );
}
```

Vous pouvez afficher une icône (FontAwesome) à gauche ou à droite du libellé.

## État désactivé et chargement

Rendez un chip inactif avec `disabled`. Affichez un indicateur avec `loading` pour les actions asynchrones.

```jsx
<div className="flex flex-wrap gap-3">
  <Chip disabled>Indispo</Chip>
  <Chip loading variant="filled">Envoi...</Chip>
</div>
```

## Tailles et forme

Ajustez la taille et la forme selon la densité de votre interface.

```jsx
<div className="flex flex-wrap items-end gap-3">
  <Chip size="small">S</Chip>
  <Chip size="medium">M</Chip>
  <Chip size="large">L</Chip>
</div>

<div className="flex flex-wrap gap-3 mt-3">
  <Chip shape="rounded">Arrondi</Chip>
  <Chip shape="squared">Angulaire</Chip>
</div>
```

## Lien ou action

Si `href` est fourni, le composant peut se comporter comme un lien. Sinon, il agit comme un contrôle interactif (click/toggle).

```jsx
<div className="flex gap-3">
  <Chip onClick={() => console.log('clic')}>Action</Chip>
  <Chip href="https://udixio.dev">Lien</Chip>
</div>
```

## Accessibilité

- Les chips commutables exposent `aria-pressed` automatiquement lorsque `onToggle` est utilisé.
- Conservez une cible tactile d’au moins 48 dp et un contraste suffisant selon le thème.
- Les libellés doivent être concis et explicites.

## Placement et mise en page

- Alignez les chips sur une ou plusieurs lignes, avec retour à la ligne si nécessaire.
- Préservez au moins 8 dp d’espacement entre chips et entre les lignes.
- Les ensembles de chips à une seule ligne peuvent défiler horizontalement si besoin.
- Les libellés trop longs se tronquent en mise en page compacte.

## Bonnes pratiques de contenu

- Préférez des noms et expressions courtes.
- Pour les suggestion chips, restez idéalement sous 20 caractères.
- Évitez le jargon ; privilégiez la clarté et l’action.

## API (résumé)

- `label` | contenu textuel du chip (ou enfants)
- `variant` | style visuel : `filled`, `elevated`, `tonal`, `outlined`
- `icon`, `iconPosition` | icône optionnelle, à gauche ou à droite
- `disabled`, `loading` | états inactif/chargement
- `onClick` | action de clic (assist/suggestion)
- `onToggle`, `activated` | sélection (filter)
- `href` | transforme le chip en lien
- `size`, `shape` | dimensions et forme

Remarque : certaines capacités (par ex. suppression d’un chip d’entrée) dépendent de votre logique d’application et des props exposées par `Chip`. Adaptez selon votre cas d’usage.
